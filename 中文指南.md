# MapAnything 模型训练中文指南

本文档面向希望在本地或集群环境中复现 / 训练 **MapAnything** 模型的工程师，涵盖环境准备、数据加工、配置体系、训练执行、监控与微调等完整流程。请结合仓库根目录下的 `README.md`、`train.md` 及 `data_processing/README.md` 一起阅读。

---

## 1. 环境与硬件准备

### 1.1 推荐硬件
- **GPU**：至少 8×80GB A100/H100 用于完整 4→24 视角课程训练；单卡或少卡可用于实验或小数据集微调。
- **CPU/内存**：分布式训练建议 32+ 物理核心、256GB RAM；数据预处理阶段需较大的存储与 I/O 吞吐。
- **存储**：原始数据 + WAI 转换 + 中间元数据约数十 TB；预训练模型与日志单独规划路径。

### 1.2 Python 环境（推理/训练）
```bash
git clone https://github.com/facebookresearch/map-anything.git
cd map-anything

# 建议使用 Python 3.12
conda create -n mapanything python=3.12 -y
conda activate mapanything

# 安装核心依赖；如需额外组件可使用可选 extras
pip install -e .
# pip install -e ".[all]"      # 安装全部可选依赖
pre-commit install             # 可选：启用提交前钩子
```

### 1.3 数据加工专用环境
WAI 数据管线依赖与训练冲突，需单独环境（详见 `data_processing/README.md`）：
```bash
conda create -n wai_processing python=3.12 -y
conda activate wai_processing
pip install --no-deps .
cd data_processing/wai_processing
pip install -e .[all]          # 或按需选择 [moge] / [mvsanywhere] 等 extras
```

## 2. 数据准备流程

### 2.1 获取原始数据集
仓库默认支持 13 个公开数据集（`train.md`- Dataset Coverage 一节列出）。每个数据集需按照各自官网协议下载。若使用 Apache 2.0 版本模型，可按 `bash_scripts/train/main` 中对应脚本仅使用可商用数据子集。

### 2.2 转换为 WAI 格式
1. 在 `wai_processing` 环境中执行各数据集 conversion：
   ```bash
   python -m wai_processing.scripts.conversion.<dataset_name> \
       original_root=/path/to/raw_dataset \
       root=/path/to/wai_dataset
   ```
2. 根据数据特性追加可选阶段（去畸变、渲染、共视、MoGe、MVSAnywhere 等），命令模板见 `data_processing/README.md`。
3. 批量调度可使用 SLURM Launcher：
   ```bash
   python -m wai_processing.launch.slurm_stage \
       data_processing/wai_processing/configs/launch/<dataset>.yaml \
       conda_env=wai_processing \
       stage=covisibility \
       launch_on_slurm=false  # 改为 true 则直接提交队列
   ```

### 2.3 下载预计算元数据
官方 Hugging Face 提供 115GB 预计算共视矩阵与数据划分，可直接复用：
```python
from huggingface_hub import snapshot_download
snapshot_download(
    repo_id="facebook/map-anything",
    repo_type="dataset",
    local_dir="/data/map-anything/metadata",
    max_workers=24,
)
```

### 2.4 验证数据可读性
使用提供的可视化脚本快速检查：
```bash
# 示例：验证 BlendedMVS 4 视图加载
python mapanything/datasets/wai/blendedmvs.py \
    --root_dir /data/wai/blendedmvs \
    --dataset_metadata_dir /data/map-anything/metadata \
    --num_of_views 4 \
    --viz
```
仅当脚本成功显示样本时再进入训练阶段，避免后续批量失败。

## 3. Hydra 配置体系总览

训练入口脚本 `scripts/train.py` 使用 Hydra 读取 `configs/train.yaml`。理解各子配置后再决定是否编写自定义配置。

### 3.1 根配置 `configs/train.yaml`
```yaml
defaults:
  - machine: aws
  - model: default
  - dataset: default
  - loss: default
  - train_params: default
  - distributed: default
  - _self_

output_dir: ${hydra:run.dir}
root_data_dir: ${machine.root_data_dir}
...
```
该文件将机器、模型、数据集、损失、训练超参等模块组合在一起。运行时可使用命令行覆盖任意字段。

### 3.2 机器配置（路径管理）
复制 `configs/machine/default.yaml` 并填写实际路径：
```yaml
root_data_dir: "/data/wai"                      # 所有 WAI 数据集合
mapanything_dataset_metadata_dir: "/data/map-anything/metadata"
root_pretrained_checkpoints_dir: "/checkpoints/pretrained"
root_experiments_dir: "/logs/mapanything"
root_uniception_pretrained_checkpoints_dir: "/checkpoints/uniception"
```
启动时通过 `machine=your_machine` 指定。若使用官方 `aws.yaml`，需保证路径与模块加载命令可用。

### 3.3 数据集配置
- 默认文件 `configs/dataset/default.yaml` 定义视角数量、分辨率选项、各子数据集引用。
- 完整 13 数据集课程训练配置示例：`configs/dataset/megatrain_13d_518_many_ar_48ipg_64g.yaml`。其中 `train_dataset` 字段采用 `+ <样本数> @ <dataset_string>` 形式对多个数据集进行加权采样。
- 若仅训练单数据集，可参考 `bash_scripts/train/examples/mapa_img_only_4v_bmvs_48ipg_8g.sh` 使用 `dataset=blendedmvs_wai_...` 一类的轻量配置。

### 3.4 模型配置
`configs/model/mapanything.yaml` 组合 encoder、信息交换层与预测头：
- `model.encoder`：默认 `dinov2_large`，可覆写以切换视觉骨干。
- `model.info_sharing`：例如 `aat_ifr_24_layers` 控制 Transformer 堆叠与跨视交互。
- `model.pred_head`：`dpt_pose_scale` 输出点云、深度、相机参数等。
- `model/task`：`images_only` / `aug_training` 等决定训练时输入模态。

若需加载预训练权重，在命令中覆写 `model.pretrained=/path/to/ckpt`.

### 3.5 训练超参数
`configs/train_params/default.yaml` 管理学习率、AMP、梯度累积、日志频率等：
- `max_num_of_imgs_per_gpu` 与 GPU 显存直接相关。
- `accum_iter` 可用于扩大有效 batch size。
- `amp_dtype` 默认为 `bf16`，无支持时自动回退。
- `save_freq`、`keep_freq` 控制 checkpoint 写入；`resume=true` 会自动从 `checkpoint-last.pth` 续训。

### 3.6 分布式设置
`configs/distributed/default.yaml` 默认单机。如需多机自定义，可覆写：
```yaml
distributed: true
world_size: 64
dist_url: tcp://<master_ip>:<port>
gpu: 0  # 每进程本地 GPU id
```
多机训练建议使用提供的 bash 脚本统一传入 rendezvous 参数。

## 4. 训练指令与脚本

### 4.1 单机多卡示例
以下示例在单机 8 卡上启动 4 视角课程阶段：
```bash
export CUDA_VISIBLE_DEVICES=0,1,2,3,4,5,6,7
export PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True  # 推荐开启

torchrun --nproc_per_node=8 scripts/train.py \
    machine=your_machine \
    dataset=megatrain_13d_518_many_ar_48ipg_64g dataset.num_views=4 \
    model=mapanything model/task=aug_training \
    model.encoder.gradient_checkpointing=true \
    train_params=lower_encoder_lr_64g \
    train_params.max_num_of_imgs_per_gpu=48 \
    hydra.run.dir="${root_experiments_dir}/mapanything/training/run_$(date +%Y%m%d_%H%M)"
```
命令行参数等价于在 YAML 中覆写，方便快速实验。

### 4.2 使用官方 Bash 脚本
`bash_scripts/train/main` 存放 Stage-1 / Stage-2 主干脚本。例如：
```bash
# 4 视角阶段（13 数据集版本）
bash bash_scripts/train/main/mapa_curri_4v_13d_48ipg_64g.sh \
     NUM_GPUS NUM_NODES NODE_RANK JOB_ID HOST_NODE_ADDR MAX_RESTARTS

# 24 视角阶段
bash bash_scripts/train/main/mapa_curri_24v_13d_48ipg_64g.sh ...
```
脚本内部会加载 NCCL/EFA 模块、设置 AWS 多机环境变量并调用 `torchrun`。在自有集群上使用时请按照脚本提示修改 `module load` 以及 Hydra 参数（尤其是 `machine=aws` 路径）。

### 4.3 多机训练关键变量
- `NUM_GPUS`：每台机器 GPU 数。
- `NUM_NODES` / `NODE_RANK`：集群规模与当前节点序号。
- `JOB_ID`、`HOST_NODE_ADDR`：`torchrun` rendezvous 相关。
- `MAX_RESTARTS`：容错重试次数。
- 环境变量中 `OMP_NUM_THREADS`、`FI_*`、`NCCL_*` 针对 EFA / IB 网络优化，可根据集群情况调优。

## 5. 训练过程监控与故障恢复

- **日志与 TensorBoard**：Hydra 会在 `hydra.run.dir` 下创建输出目录，包含 `log.txt`、`events.out.tfevents.*`。启动 TensorBoard：
  ```bash
  tensorboard --logdir /logs/mapanything --bind_all
  ```
- **检查点策略**：
  - `checkpoint-last.pth`：每轮保存，可用于自动续训。
  - `checkpoint-XX.pth`：按照 `keep_freq` 间隔保留的历史版本。
  - `checkpoint-best.pth`：`Average Test Loss Median` 最优模型。
  - `checkpoint-final.pth`：训练结束写入。
- **评估频率**：`train_params.eval_freq` 控制验证周期。测试集由 `dataset.test_dataset` 中的加权列表决定。
- **异常排查**：
  - 若某数据集 loader 失败，优先在单机上执行相应 `mapanything/datasets/wai/<dataset>.py --viz`。
  - 若显存不足，降低 `train_params.max_num_of_imgs_per_gpu` 或启用 `model.info_sharing.module_args.gradient_checkpointing=true`、`model.pred_head.gradient_checkpointing=true`。
  - 对于自定义 Hydra 配置，使用 `python scripts/train.py --cfg job --resolve` 检查最终展开结果。

## 6. 微调与扩展

- `bash_scripts/train/finetuning/` 提供 VGGT、MoGe-2、π³ 微调脚本，可直接传入 GPU 数运行。

## 7. 常见注意事项与建议

- **路径与权限**：确保 `root_experiments_dir` 可写；分布式训练建议共享文件系统以便所有节点访问 checkpoint。
- **随机种子**：默认 `train_params.seed=0`，多机时内部会根据 rank 偏移；如需完全确定性可额外设置 `CUBLAS_WORKSPACE_CONFIG=:16:8` 等环境变量。
- **混合精度**：默认开启 AMP(`bf16`)；在部分 GPU 上需手动切换 `train_params.amp_dtype="fp16"`。
- **恢复训练**：若需从特定 checkpoint 重启，可显式设置 `train_params.resume_ckpt=/path/to/ckpt.pth`。
- **自定义数据集**：在 `data_processing/wai_processing/scripts/conversion/` 下新增转换脚本，并在 `configs/dataset` 配置中引用。
- **版本控制**：仓库提供 `pre-commit`，在提交前自动运行格式化和 lint。

## 8. 训练完成后的评估与导出

1. **Benchmark**：`benchmarking/` 目录含多项评测 README。按照对应说明运行 `bash_scripts/benchmarking/...` 或 `python benchmarking/...`。
2. **转换 Hugging Face 检查点**：若需与官方格式保持一致，可使用 `scripts/convert_hf_to_benchmark_checkpoint.py` 输出包含 `model` 键的权重文件。
3. **COLMAP / GSplat 集成**：参照 `README.md` 中的 COLMAP 章节，运行 `scripts/demo_colmap.py` 导出稀疏重建，并可进一步用于 Gaussian Splatting。

## 9. 全流程速查清单

1. **克隆仓库 & 创建 Python 环境**（`conda create -n mapanything python=3.12`，`pip install -e .`）。
2. **准备数据**：下载原始集 → 使用 `wai_processing` 环境转换 → 下载 Hugging Face 元数据 → `--viz` 验证。
3. **配置 Hydra**：复制 `configs/machine/default.yaml` 填写路径；选择/编写数据集与模型配置；确认训练超参。
4. **启动训练**：单机使用 `torchrun` 或直接调用 `bash_scripts/train/main`；根据 GPU 规模调整 `max_num_of_imgs_per_gpu` 与 `accum_iter`。
5. **监控与调优**：查看 TensorBoard、`log.txt`、定期验证指标；必要时调整损失权重或采样策略。
6. **保存与续训**：记录 `hydra.run.dir`，确保 `checkpoint-best.pth`、`checkpoint-final.pth` 等可复现；如需继续训练或微调，直接在命令中指定 `train_params.resume_ckpt`。
7. **评测与发布**：利用 `benchmarking/` 脚本验证性能，按需转换权重并集成下游应用。

按照以上步骤，即可在本地或集群环境中高效、可重复地训练 MapAnything 模型。祝顺利！
